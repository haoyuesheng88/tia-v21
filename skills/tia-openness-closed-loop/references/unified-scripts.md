# Unified Scripts

This reference captures the validated `WinCC Unified V21` script workflow from a live project.

## What Was Verified

- `HMI_RT_2` exposes `Scripts` as `Siemens.Engineering.HmiUnified.Scripts.HmiScriptModuleComposition`
- The composition supports:
  - `Import(DirectoryInfo sourceLocation)`
  - `Import(DirectoryInfo sourceLocation, string name)`
  - `Export(DirectoryInfo destinationLocation)`
- A real imported module appeared in the project tree and could be exported back out

## Known-Good Import Package Format

Use a directory containing exactly these files:

```text
ClosedLoopModule.hmi.yml
ClosedLoopModule.hmi.js
```

Example `ClosedLoopModule.hmi.yml`:

```yml
#Version: 2.0

ScriptModules:
  ClosedLoopModule:
    ScriptFile: ClosedLoopModule.hmi.js
```

Example `ClosedLoopModule.hmi.js` structure:

```javascript
/*<auto-generated>
***End of Global definition area***
Changes to this comment may cause incorrect behavior during document based import.
</auto-generated>*/

export function StartPulse() {
  // ...
}
```

Preserve the autogenerated header block. Import is sensitive to structure.

## Recommended Closed-Loop Functions

Use these exported function names so later automation can rely on stable names:

- `StartPulse`
- `StopPulse`
- `WriteSetValue`
- `StartWithPreset`
- `ReadClosedLoopState`

## Recommended Tag Set

- `HMI_Start_PB`
- `HMI_Stop_PB`
- `HMI_Run_Cmd`
- `HMI_Lamp_FB`
- `HMI_SetValue`
- `HMI_ActualValue`

## Closed-Loop Behavior

- `StartPulse` writes `HMI_Start_PB = true`, then resets it after `200 ms`
- `StopPulse` writes `HMI_Stop_PB = true`, then resets it after `200 ms`
- `WriteSetValue` writes `HMI_SetValue`
- `StartWithPreset` batch-writes `HMI_SetValue` and `HMI_Start_PB`
- `ReadClosedLoopState` uses `CreateTagSet(...).Read(...)` and `HMIRuntime.Trace(...)`

## Recommended Follow-Up After Import

1. Re-enumerate script modules and confirm the module name exists.
2. Export the module back out and compare the `.js` content.
3. Bind screen events or tasks to the imported functions.
4. Verify the PLC still owns the latch and numeric feedback logic.

## Important Limits

- Importing a script module does not automatically bind buttons, IO fields, or tasks.
- A script module alone is not a full HMI closed loop until screen objects call the functions.
- Keep the process logic in PLC `SCL`. Use Unified `JavaScript` for pulse generation, batch writes, and feedback reads.
